# QA funcional — Dental Pilot v0.7.3

## Objetivo
Corregir la inconsistencia entre el conteo de pacientes en estado CRM Agendado y la sección Próximas citas del dashboard.

## Validaciones realizadas

### Dashboard
- [x] La métrica principal cambió de “Pacientes agendados” a “Citas próximas”.
- [x] “Citas próximas” cuenta solamente citas futuras activas.
- [x] “Próximas citas” excluye citas finalizadas, canceladas y no asistidas.
- [x] Se agregó una métrica separada “Agendados sin cita”.
- [x] Si existen pacientes en CRM Agendado sin cita futura activa, se muestra una alerta explicativa.
- [x] La alerta permite navegar a Pacientes para revisión.

### Reglas funcionales
- [x] Paciente con estado CRM Agendado sin cita futura no aparece en Próximas citas.
- [x] Paciente con cita futura activa sí aparece en Próximas citas.
- [x] Citas cerradas no aparecen como próximas.

### Regresión
- [x] No se cambió la estructura de base de datos.
- [x] No se modificaron flujos de agenda, pacientes, servicios, tratamientos ni presupuestos.
- [x] Se actualizó la versión global a v0.7.3.

## SQL
No requiere SQL nuevo.

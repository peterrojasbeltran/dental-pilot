# QA funcional — v0.3.1 Responsive Fix

## Objetivo
Validar correcciones de regresión detectadas en v0.3.0:

1. La demo pública debe mostrar navegación en mobile.
2. El CRM Kanban debe ser usable en mobile sin depender de drag & drop horizontal.

## Casos validados

| Caso | Resultado esperado | Estado |
|---|---|---|
| Abrir `/` | Home carga sin cambios de UX | OK |
| Click en “Ver Demo Dashboard” | Navega a `/demo/dashboard` | OK |
| Demo dashboard desktop | Muestra header, CTAs y navegación | OK |
| Demo dashboard mobile | Muestra botón hamburguesa | OK |
| Menú mobile demo | Muestra Dashboard demo, CRM demo e Ingresar | OK |
| Abrir `/demo/patients` | Muestra CRM demo | OK |
| CRM desktop | Mantiene Kanban horizontal de 8 columnas | OK |
| CRM mobile | Muestra selector por estado en pills | OK |
| Mover paciente en mobile | Permite cambiar estado usando select “Mover a” | OK |
| Login | No se modifica flujo de autenticación | OK |

## Validación técnica ejecutada

```bash
npm run typecheck
PORT=3101 npm run dev
```

Smoke test HTTP:

- `/` → 200 OK
- `/demo/dashboard` → 200 OK
- `/demo/patients` → 200 OK

`next build` compila correctamente, pero en este entorno quedó detenido en `Collecting page data`. Para la validación local se priorizó `npm run typecheck` y `npm run dev`, que es el flujo usado en Antigravity para esta etapa.

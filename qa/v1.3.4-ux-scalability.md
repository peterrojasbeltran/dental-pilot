# QA Funcional — v1.3.4 UX Scalability

## Objetivo
Validar que Dental Pilot sea usable con datasets grandes.

## Casos principales

### Pacientes
- PASS esperado: lista con paginación 10/25/50/100.
- PASS esperado: búsqueda reinicia a página 1.
- PASS esperado: KPIs operativos visibles.

### Tratamientos
- PASS esperado: lista compacta mantiene filtros y paginación.
- PASS esperado: detalle expandible funciona con página actual.

### Presupuestos
- PASS esperado: lista compacta con paginación y filtros por estado.
- PASS esperado: detalle por fila no rompe mobile.

### Inventario
- PASS esperado: tabla/cards paginadas.
- PASS esperado: filtro de stock bajo reinicia a página 1.

### Finanzas
- PASS esperado: egresos recientes paginados.
- PASS esperado: KPIs de egresos y categorías visibles.

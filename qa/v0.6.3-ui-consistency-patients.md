# QA Funcional v0.6.3 — Consistencia visual Pacientes

## Objetivo
Validar que Pacientes use el mismo patrón compacto de acciones que Servicios.

## Casos de prueba

### 1. Acciones compactas
- Ir a Pacientes.
- Verificar que cada tarjeta muestre botón Editar e ícono Activar/Desactivar juntos.
- Verificar que no aparezca el texto largo “Desactivar paciente”.

### 2. Badge Activo/Inactivo
- Verificar que el estado se muestre como badge pequeño.
- Desactivar un paciente y validar que cambie a Inactivo.
- Activar el paciente nuevamente y validar que cambie a Activo.

### 3. Mobile
- Abrir Pacientes en ancho móvil.
- Verificar que las acciones no ocupen espacio excesivo.
- Verificar que editar paciente siga abriendo el panel móvil correctamente.

### 4. Regresión
- Crear paciente.
- Editar paciente.
- Validar teléfono duplicado con toast.
- Verificar Servicios y Agenda sin cambios visibles inesperados.

## SQL
No requiere SQL nuevo.

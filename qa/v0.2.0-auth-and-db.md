# QA funcional - v0.2.0 Auth + DB

## Instalación

- [ ] Ejecutar `npm install`.
- [ ] Crear `.env.local` desde `.env.example`.
- [ ] Ejecutar `npm run dev`.
- [ ] Abrir `http://localhost:3000`.

## Supabase

- [ ] Ejecutar `supabase/schema.sql` en SQL Editor.
- [ ] Crear usuario `admin@dentalpilot.test`.
- [ ] Confirmar que existen datos en `clinic_settings`, `patients`, `services`, `appointments`.

## Login

- [ ] Abrir `/login`.
- [ ] Ingresar email y password válidos.
- [ ] Redirige a `/dashboard`.
- [ ] Botón salir vuelve a `/login`.
- [ ] Credenciales incorrectas muestran mensaje de error.

## Rutas protegidas

- [ ] Sin sesión, `/dashboard` redirige a `/login` cuando Supabase está configurado.
- [ ] Con sesión, `/dashboard` carga correctamente.

## Dashboard

- [ ] Muestra país Bolivia.
- [ ] Muestra moneda Bs.
- [ ] Muestra citas próximas.
- [ ] Muestra servicios activos.

## Pacientes

- [ ] `/patients` carga listado.
- [ ] Muestra estado y riesgo.
- [ ] No rompe en mobile.

## Servicios

- [ ] `/services` muestra precios en Bs.
- [ ] Muestra duración de servicios.

## Agenda

- [ ] `/appointments` muestra citas base.
- [ ] Muestra fecha/hora con locale configurado.

## Responsive

- [ ] Landing usable en móvil.
- [ ] Login usable en móvil.
- [ ] Dashboard usable en móvil.
- [ ] Tablas permiten scroll horizontal cuando aplica.
